# 예시 ADR

## ADR-UI-001: 다크 모드 기본 지원

### 메타데이터

| 항목 | 내용 |
|------|------|
| 날짜 | 2024-01-15 |
| 상태 | Accepted |
| 결정자 | 개발팀 |
| 영향 범위 | 전체 |

### 컨텍스트

**문제 상황:**
- 타겟 사용자(개발자, 디자이너)가 다크 모드를 선호
- 경쟁사(Linear, Vercel, GitHub) 모두 다크 모드 지원
- 사용자 피드백에서 다크 모드 요청 다수

**제약 조건:**
- Tailwind CSS 사용 중
- 기존 색상 변수 재정의 필요

### 고려한 옵션

**옵션 1: 라이트 모드만**
- 장점: 구현 단순
- 단점: 사용자 기대 미충족

**옵션 2: 다크 모드만**
- 장점: 타겟 사용자 선호
- 단점: 일부 사용자 불만

**옵션 3: 시스템 설정 + 수동 토글**
- 장점: 모든 사용자 만족
- 단점: 구현 복잡

### 결정

**선택:** 옵션 3 - 시스템 설정을 기본으로 따르되, 수동 토글 제공

**이유:**
- 사용자 선호 존중
- 접근성 향상
- 경쟁사 대비 동등 기능

### 구현 세부사항

```tsx
// tailwind.config.js
module.exports = {
  darkMode: 'class',
  // ...
}

// ThemeProvider
<ThemeProvider attribute="class" defaultTheme="system">
  {children}
</ThemeProvider>
```

### 결과

**예상 영향:**
- 사용자 만족도 향상
- 추가 개발 시간 약 2시간

**측정 지표:**
- 다크 모드 사용 비율 추적
- 관련 피드백 모니터링

## ADR-UI-002: Skeleton UI 표준화

### 메타데이터

| 항목 | 내용 |
|------|------|
| 날짜 | 2024-01-16 |
| 상태 | Accepted |
| 결정자 | 개발팀 |
| 영향 범위 | 전체 데이터 로딩 컴포넌트 |

### 컨텍스트

**문제 상황:**
- 로딩 상태 UI가 일관되지 않음
- 일부는 Spinner, 일부는 빈 화면
- CLS(Cumulative Layout Shift) 점수 저하

### 결정

**선택:** 모든 데이터 로딩에 Skeleton UI 사용

**구현:**
- Skeleton, CardSkeleton, ListItemSkeleton 등 표준 컴포넌트 사용
- Spinner는 버튼 내부 로딩에만 사용

### 결과

**긍정적 영향:**
- CLS 점수 개선
- 체감 로딩 속도 향상
- UI 일관성 확보
