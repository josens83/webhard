# EduVault - Project Vision

## One-Line Definition
A comprehensive educational cloud storage platform that enables secure file management, real-time collaboration, and seamless content sharing for educational institutions.

## Target Users
1. **Students**: Store and access course materials, submit assignments, collaborate with peers
2. **Educators**: Distribute materials, manage class resources, communicate with students
3. **Administrators**: Manage users, monitor storage usage, oversee platform operations

## Problems We Solve
1. Fragmented file storage across multiple platforms
2. Difficulty in sharing large educational files securely
3. Lack of real-time communication integrated with file sharing
4. Complex permission management for educational content
5. No unified platform for Korean educational institutions

## Core Values (Maximum 3)
1. **Security First** - Enterprise-grade file protection and access control
2. **Simplicity** - Intuitive interface that requires minimal training
3. **Reliability** - 99.9% uptime with automatic backups

## Core Features (MVP)
- [x] User authentication and authorization
- [x] File upload, download, and management
- [x] Folder hierarchy and organization
- [x] Secure file sharing with expirable links
- [x] Real-time messaging between users
- [x] Storage quota management
- [x] Admin dashboard for user management
- [x] Payment integration for premium storage

## Target NOT (Explicitly Out of Scope)
- Video conferencing/live streaming
- Document editing (Google Docs-like features)
- LMS (Learning Management System) features
- Mobile native apps (web responsive only)
- Multi-language support (Korean only for MVP)
- Third-party integrations (Slack, Teams, etc.)
- AI-powered features (search, recommendations)

## Success Metrics
| Metric | Target | Measurement |
|--------|--------|-------------|
| Page Load Time | < 2 seconds | Lighthouse Performance Score > 80 |
| File Upload Speed | < 5 seconds per 100MB | Backend monitoring |

## Technical Boundaries
- Maximum file size: 5GB per file
- Storage quota: Based on subscription tier
- Concurrent connections: 10,000 users
- Supported file types: All common formats
